Intro2C for XBASE Developers
Data Typing
 
 
We've become lazy with xbase. :-) We can use all data types with a single variable name, however we need. We have  TYPE()  to tell us what type of data is current contained within the variable. In C, the type is set in its definition and cannot later be changed.

For example, in xbase the following is legal:

  
k = 5
k = "Hello"
k = .f.
 

In C, it's not variable. Each type is specific and explicit. This allows the compiler to utilize machine resources in the most efficient manner. It's able to know exactly how something will be used, so it can allocate what's needed to make it happen. And with optimization abilities, the compiler can sometimes completely change the expression of the source code so that internally it runs completely differently than you (the developer) indicated, and yet it still accomplishes the same goals ... just much faster.

C is currently the fastest language available. It is because it has all of these compile-time determinations that it can rely upon.
